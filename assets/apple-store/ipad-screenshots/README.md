# iPad Screenshots

Place your iPad screenshots in this folder with the following naming convention:

## 12.9" Display (iPad Pro 12.9", 6th gen)
**Required - Priority 1**
- Portrait: 2048 × 2732 pixels
- Landscape: 2732 × 2048 pixels
- Files:
  - `ipad-12.9-01-home.png`
  - `ipad-12.9-02-quran.png`
  - `ipad-12.9-03-azkar.png`
  - `ipad-12.9-04-prayer-times.png`
  - `ipad-12.9-05-ramadan.png`

## 11" Display (iPad Pro 11", iPad Air)
**Optional but Recommended**
- Portrait: 1668 × 2388 pixels
- Landscape: 2388 × 1668 pixels
- Files:
  - `ipad-11-01-home.png`
  - `ipad-11-02-quran.png`
  - etc.

## 10.5" Display (iPad Pro 10.5", iPad Air 3rd gen)
**Alternative Option**
- Portrait: 1668 × 2224 pixels
- Landscape: 2224 × 1668 pixels

## 9.7" Display (iPad, iPad Pro 9.7")
**Optional - Legacy Support**
- Portrait: 1536 × 2048 pixels
- Landscape: 2048 × 1536 pixels

## iPad-Specific Screenshot Guidelines

### Content Recommendations
iPad has more screen real estate, so showcase:
1. **01-home**: Split-view or expanded dashboard
2. **02-quran**: Side-by-side pages or with translation
3. **03-azkar**: Grid view or larger layout
4. **04-prayer-times**: Calendar view or expanded schedule
5. **05-ramadan**: Comprehensive timetable with stats

### Design Considerations for iPad
- Take advantage of larger screen size
- Show tablet-optimized layouts (not just scaled iPhone UI)
- Consider landscape orientation for key features
- Show multitasking if app supports it
- Demonstrate any iPad-specific features

### iPad UI Patterns
- Master-detail layouts
- Split views
- Popovers for settings/options
- Wider navigation bars
- Larger touch targets
- More content visible at once

### Tools for iPad Screenshots
- **iPad Simulator**: Xcode (Window → Devices and Simulators)
- **Real iPad**: Screenshot with Volume Up + Power button
- **Fastlane**: Automated cross-device screenshots
- **Design Tools**: Scale iPhone designs appropriately

### File Format
- Format: PNG (24-bit)
- Color profile: sRGB or Display P3
- No transparency
- Maximum file size: 8 MB per screenshot

## Testing on iPad
Make sure to test your app on actual iPad or simulator to ensure:
- Layout adapts properly to larger screens
- Touch targets are appropriately sized
- Text is readable (not just scaled up)
- Navigation makes sense on larger display
- Orientation changes work smoothly
- No awkward stretching or misaligned elements
